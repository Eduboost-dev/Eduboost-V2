# Required Secrets and Environment Variables

**Owner:** Platform / Engineering
**Last updated:** 2026-05-27 (Phase 1 T144)
**Status:** Active

This document lists all secrets and sensitive environment variables required
for EduBoost V2 to run in production, staging, and CI.

---

## GitHub Secrets (`Repository secrets`)

These are stored in the GitHub repository settings (`Settings → Secrets and
variables → Actions`).

| Secret | Used by | Purpose | Sensitivity |
|---|---|---|---|
| `GITHUB_TOKEN` | All workflows | GitHub API access (auto-provided) | Low (auto-rotated) |
| `STAGING_URL` | `ci-cd.yml` | Staging base URL for smoke tests | Low |
| `STAGING_API_KEY` | `ci-cd.yml` | Staging API key for smoke tests | Medium |
| `STAGING_SMOKE_BASE_URL` | `staging-smoke.yml` | Staging URL for scheduled smoke tests | Low |
| `STAGING_LEARNER_TOKEN` | `staging-smoke.yml` | Learner JWT for staging smoke tests | Medium |
| `PROD_URL` | `ci-cd.yml` | Production health check URL | Low |
| `KUBECONFIG_DATA` | `ci-cd.yml` | Base64-encoded kubeconfig for k8s deploy | **Critical** |
| `TEST_LEARNER_PASSWORD` | `ci-cd.yml` | Test learner password for E2E tests | Low (test env only) |
| `ADMIN_API_TOKEN` | Workflows | Admin API token for privileged CI operations | **Critical** |
| `ANTHROPIC_API_KEY` | Workflows | Anthropic LLM API key | **Critical** |
| `GROQ_API_KEY` | Workflows | Groq LLM API key | **Critical** |
| `DIAG_SCORE_DATABASE_URL` | `diag-score-live-audit.yml` | Diagnostic score DB connection | **Critical** |
| `DB_ROLLBACK_SOURCE_DATABASE_URL` | `db-backup-restore-rollback-evidence.yml` | Source DB for rollback test | **Critical** |
| `AUDIT_WRITE_DATABASE_URL` | `audit-write-runtime-evidence.yml` | Audit DB connection | **Critical** |

**Total:** 15 repository secrets.

## Render Environment Variables (`render.yaml`)

These are configured in the Render dashboard or via the Blueprint (`render.yaml`).

| Variable | Source | Purpose |
|---|---|---|
| `DATABASE_URL` | `sync: false` | Supabase Postgres connection string |
| `SECRET_KEY` | `generateValue: true` | JWT signing key (auto-generated by Render) |
| `UPSTASH_REDIS_REST_URL` | `sync: false` | Upstash Redis REST endpoint |
| `UPSTASH_REDIS_REST_TOKEN` | `sync: false` | Upstash Redis REST token |
| `REDIS_URL` | `sync: false` | Standard Redis connection string |

**Note:** `sync: false` means the value is managed in the Render dashboard
and not synced from the Blueprint on subsequent deploys.

| `JWT_SECRET` | `eduboost-jwt-secret` | JWT signing key |
| `ENCRYPTION_KEY` | `eduboost-encryption-key` | PII encryption key |
| `ENCRYPTION_SALT` | `eduboost-encryption-salt` | PII encryption salt |

| `ANTHROPIC_API_KEY` | `eduboost-anthropic-api-key` | Anthropic LLM API |

## Secret rotation policy

| Secret type | Rotation frequency | Trigger |
|---|---|---|
| JWT signing key | Every 90 days | Key Vault rotation loop |
| LLM API keys | Every 90 days | Manual or vendor notification |
| Database credentials | Every 180 days | Supabase / Render rotation |
| Kubeconfig | Every 180 days | k8s cluster admin |
| Azure Key Vault | Managed by Azure | Automatic (optional) |

**Procedure:**
1. Generate new secret.
2. Update in secret store (GitHub / Render / Key Vault).
3. Trigger `emergency_revoke_all()` for JWT keys.
4. Verify application health.
5. Revoke old secret after 24-hour grace period.

## Validation

```bash
# List all secrets referenced in workflows
grep -roh "secrets\.[A-Z_]*" .github/workflows/ | sort | uniq

# Confirm no literal secrets in source code
grep -rn "JWT_SECRET\|SECRET_KEY\|ENCRYPTION_KEY" app/ --include="*.py" | grep -v "settings\." | grep -v "os\.getenv"
# Should return only the config.py defaults (placeholders)
```
