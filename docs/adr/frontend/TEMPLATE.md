# Architecture Decision Record Template (Frontend)

```text
ADR Title: <short summary>
Status: Draft | Proposed | Accepted | Superseded
Date: YYYY-MM-DD
Owner: Role or name
RC Gate: RC1 | RC2 | RC3 | RC4 | RC5
Blocks: Task IDs blocked by this ADR
Reviewers: Roles that must sign off (e.g., Tech Lead, POPIA Compliance Officer)
Evidence: Spike report, experiments, or references used to make this decision
```

## Context

Describe the problem statement, constraints, and any research or spikes that inform the decision.

## Decision

State the decision in clear, testable terms. Prefer bullet points when multiple sub-decisions exist.

## Consequences

### Positive

- List benefits and what the team gains by following this ADR.

### Negative

- List trade-offs, risks, or work created by the decision.

## Implementation Notes

Document roll-out plans, sequencing, or tooling changes needed to put the decision into practice.

## Compliance & Evidence

Reference POPIA, security, or product reviews plus links to spike reports, PRs, or test evidence.
