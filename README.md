## Documentation Structure

All project documentation lives under [`docs/`](docs/) with domain-specific subdirectories:

- `docs/roadmap/` — Roadmap, process discipline, execution plans, and implementation reports
- `docs/todos/` — North Star TODO and task tracking
- `docs/release/` — Phase evidence, audits, release bundles
- `docs/adr/` — Architecture Decision Records
- `docs/architecture/` — Architecture reports and router audits
- `docs/operations/` — Startup guides, runbooks
- `docs/caps/` — CAPS curriculum alignment
- `docs/curriculum/` — Study material expansion plans
- `docs/content_factory/` — Content generation configuration
- `docs/security/` — Security posture and threat models
- `docs/popia/` — POPIA compliance
- …and 40+ other specialized directories.
